## Directory Purposes

**`backend/`:**
- Purpose: Python server, all render pipelines, storage
- Key files: `app.py` (all routes + job queue), `saasshorts.py` (UGC), `content.py` (philosophical)

**`backend/output/`:**
- Purpose: Per-job rendered artifacts — `output/{job_id}/` subdirectory per job
- Structure: `{title_slug}_actor.png`, `{title_slug}_voice.mp3`, `{title_slug}_head.mp4`, `{title_slug}_subs.ass`, `{title_slug}_final.mp4`; content pipeline: `voiceover.mp3`, `bg_0.png`…, `bg_clip_0.mp4`…, `subs.ass`, `content_video.mp4`
- Also contains `metadata.json` for gallery entries when S3 is unavailable
- Served at `/videos/` by FastAPI `StaticFiles` mount

**`backend/uploads/`:**
- Purpose: Actor portrait images generated by fal.ai or user-uploaded
- Served at `/uploads/` by FastAPI `StaticFiles` mount

**`backend/assets/music/`:**
- Purpose: Ambient MP3 tracks for the content pipeline
- Generated: Yes — synthesized via FFmpeg at first import of `content.py`
- Committed: No — regenerated if spec version bumps

**`backend/static/`:**
- Purpose: Production frontend build (copied from `frontend/dist/`)
- Served by SPA fallback catch-all route at `backend/app.py:725`
- Not present in dev (Vite serves directly)

**`frontend/src/pages/`:**
- Purpose: One file per route; each page owns its step state machine
- Pattern: `useState(0)` step counter + per-step result state passed down to step components

**`frontend/src/components/product/`** and **`frontend/src/components/content/`:**
- Purpose: Step-level UI components; receive data via props, emit completion via `onComplete` callback
- Each step component is self-contained — imports from `lib/api.js` directly

**`frontend/src/lib/`:**
- Purpose: Pure utility modules with no React dependencies
- `api.js`: all backend fetch calls, custom headers injected from `keys` arg
- `storage.js`: thin `localStorage` wrappers

## Naming Conventions

**Files (backend):**
- Snake_case Python modules: `saasshorts.py`, `s3_uploader.py`, `content.py`
- Private helpers prefixed with `_`: `_fal_run`, `_run_ugc_job`, `_composite_content_video`

**Files (frontend):**
- PascalCase React components: `ProductPage.jsx`, `AnalyzeStep.jsx`, `VideoCard.jsx`
- camelCase utility modules: `api.js`, `storage.js`, `useApiKeys.js`

**Job types:**
- `type="ugc"` (default, omitted from job dict for legacy jobs) — product UGC pipeline
- `type="content"` — philosophical content pipeline
- Discriminated in `run_job` at `backend/app.py:278`

**Output file naming (UGC):**
- `{title_slug}_{role}.{ext}` — e.g., `my_saas_product_actor.png`, `my_saas_product_final.mp4`
- `title_slug` is derived from script title: lowercase alphanumeric + underscores, max 30 chars (`saasshorts.py:1350`)

**Output file naming (content):**
- Fixed names: `voiceover.mp3`, `bg_{i}.png`, `bg_clip_{i}.mp4`, `subs.ass`, `content_video.mp4`

## Where to Add New Code

**New pipeline step (UGC):**
- Add function to `backend/saasshorts.py` following the `generate_*` naming pattern
- Call from `generate_full_video` (`saasshorts.py:1320`)
- Add progress log calls via the `log` callback

**New pipeline step (content):**
- Add function to `backend/content.py`
- Call from `generate_content_video` (`content.py:673`)

**New API endpoint:**
- Add Pydantic model inline in `backend/app.py` (before `# --- Endpoints ---` at line 122)
- Add route handler in `app.py` in the appropriate domain section (`# --- Content endpoints ---` or `# --- Background workers ---`)
- Add corresponding fetch function in `frontend/src/lib/api.js`

**New frontend page:**
- Add page component in `frontend/src/pages/`
- Add step components in `frontend/src/components/{domain}/`
- Register route in `frontend/src/App.jsx`

**New third-party service integration:**
- Backend: add to `backend/app.py` env var reads + header extraction pattern; call via `run_in_executor`
- Frontend: add key to `KEY_NAMES` in `frontend/src/hooks/useApiKeys.js`; add header in `frontend/src/lib/api.js:headers()`

**Shared pipeline utilities:**
- Helpers used by both pipelines belong in `backend/saasshorts.py` (which `content.py` imports from)
